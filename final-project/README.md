# Final Project: Build Your Own Website

## Overview

Congratulations on completing all three parts of this course! Now it's time to put everything together and build a complete, interactive website from scratch.

## Project Description

Create a multi-page personal website (or choose one of the suggested projects below) that demonstrates your mastery of HTML, CSS, and JavaScript.

## Requirements

Your website must include:

### HTML Requirements
- [ ] Proper semantic HTML structure (header, nav, main, sections, footer)
- [ ] At least 3 pages with consistent navigation
- [ ] A form with validation
- [ ] Appropriate use of headings, lists, links, and images
- [ ] All images must have alt text

### CSS Requirements
- [ ] External stylesheet(s) with organized code
- [ ] Responsive design that works on mobile, tablet, and desktop
- [ ] Use of Flexbox and/or Grid for layout
- [ ] Custom color scheme and typography
- [ ] Hover effects and transitions
- [ ] At least one animation
- [ ] Consistent styling across all pages

### JavaScript Requirements
- [ ] Interactive navigation (e.g., mobile menu toggle)
- [ ] Form validation with helpful error messages
- [ ] At least 3 interactive features (e.g., image slider, accordion, modal, tabs)
- [ ] Dynamic content manipulation
- [ ] Event listeners for user interactions
- [ ] Smooth scrolling or other UX enhancements

## Evaluation Criteria

Your project will be evaluated on:

- **Functionality** (30%): Does everything work as expected?
- **Design** (25%): Is it visually appealing and well-organized?
- **Code Quality** (20%): Is your code clean, organized, and commented?
- **Responsiveness** (15%): Does it work well on all screen sizes?
- **Creativity** (10%): Did you add unique features or designs?

## Bonus Features

- [ ] Local storage for saving user preferences
- [ ] API integration (e.g., weather, quotes, news)
- [ ] Progressive Web App features

- [ ] SEO meta tags

## Tips for Success

1. **Start Simple**: Get a basic version working first, then add features
2. **Use Version Control**: Consider using Git to track your changes
3. **Test Early and Often**: Don't wait until the end to test
4. **Mobile First**: Design for mobile devices first, then scale up
5. **Keep it Organized**: Use clear file structure and naming conventions
6. **Comment Your Code**: Help your future self understand what you built
7. **Ask for Feedback**: Show others and get their input
8. **Be Creative**: Add your personal touch and style
